    《AI 大模型训练与推理建模仿真白皮书》大纲 ¶

    建模仿真概述 ¶

    全生命周期的数字化底座 ¶

    • 规划期：架构选型与理论收益预估
    • 建设期：策略验证与参数预调优
    • 维护期：故障演练与鲁棒性分析
    • 优化期：存量资源效能挖掘

    核心场景 ¶

    • 性能推演（Performance Projection）
    • 假设分析（What-if Analysis）
    • 快速评估与优化（Rapid Evaluation & Optimization）

    方法论体系 ¶

    • 客观映射体系：Input Space \(\to\) System Behavior
    • 仿真维度递进：功能 \(\to\) 性能 \(\to\) 行为
    • 层次化架构：负载（需求）- 系统（映射）- 资源（供给）

    负载建模（Workload Modeling）：算力需求定义 ¶

    核心职能：构建平台无关的中间表示（IR），准确描述算法逻辑的资源需求。

    定义与表达形态（Definition & Abstraction Forms）¶

    • 核心定义：连接算法与基建的数学表达。
    • 形态谱系：
      • 静态计算图（Static DAG）：面向结构确定的训练负载。
      • 动态执行迹（Dynamic Execution Trace）：面向 MoE、Agent 及推理等非确定性负载。
    • 建模粒度：模型级（Model） / 算子级（Kernel） / 指令级（Instruction）。

    负载特征分类（Workload Classification）¶

    • 确定性负载：编译期确定的图结构（如 ResNet, Standard LLM Training）。
    • 动态路由负载：依赖运行时数据的条件分支（如 MoE Top-k Gating）。
    • 数据依赖型负载：随输入分布波动的计算量（如 Variable Sequence Length, Sparse Attention）。

    正交语义描述体系（Orthogonal Semantics）¶

    • 计算语义：
      • 区分稠密计算（Prefill）与访存密集型计算（Decode）。
      • 算子类型、FLOPs 与精度约束。
    • 通信语义：
      • 逻辑意图：全局集合通信（Collective Intent）与点对点依赖（P2P Dependency）。
      • 解耦原则：不预设物理路径与具体算法（Ring/Tree）。
    • 存储与访存语义：
      • 生命周期管理：参数、激活值与 KV Cache 的 Alloc/Free 时序。
      • 访问模式：连续访存 vs. 非连续 / 碎片化访存（Paged Memory）。
    • 时序语义：基于数据依赖的因果序（Causality）与逻辑屏障。

    负载驱动的系统仿真（System Simulation）：映射与编排 ¶

    核心职能：System Layer 作为仿真引擎的“编排器（Orchestrator）”，负责将逻辑负载映射到物理资源，并管理计算、通信与存储的动态执行流。

    映射空间与原语转换（Mapping & Lowering）¶

    定位：定义从“逻辑意图”到“物理指令”的静态转换规则，是仿真的空间配置阶段。

    • 3.1.1 逻辑 - 物理映射（Logical-Physical Mapping）

      • 算子放置（Placement）：定义逻辑计算图节点到物理设备坐标（Rank/Rack/Node/GPU ID）的绑定关系（Device Map）。
      • 并行策略实例化（Parallelism Instantiation）：
        • 切分（Tiling）：依据 TP/PP/DP 策略，将单体算子切分为物理设备上的 Sub-kernels。
        • 胶水逻辑生成：在切分边界自动插入必要的 Send/Recv 或 Collective 通信节点。

    • 3.1.2 通信原语降级（Communication Lowering）

      • 算法展开（Algorithm Expansion）：将高层集合通信意图（如 AllReduce）基于物理拓扑（如 Torus/Fat-tree/NVLink）展开为具体的 P2P 传输序列（如Ring vs Tree vs Mesh算法）。
      • 数据分块（Chunking）：定义大张量传输的切块策略（Chunk Size），以模拟细粒度的计算通信重叠（Overlap）。

    • 3.1.3 资源仲裁配置（Resource Arbitration Config）

      • 流映射（Stream Mapping）：定义计算流、通信流（H2D/D2H/P2P）在硬件队列上的优先级与并发规则。
      • 协议开销模拟：定义跨节点通信的握手延迟与协议开销（Protocol Overhead）。

    训练场景仿真：同步流水线编排 ¶

    定位：基于 3.1 的映射规则，模拟训练场景下强同步、周期性、批处理的执行流。

    • 3.2.1 流水线调度（Pipeline Scheduling）

      • 微批次编排（Micro-batch Orchestration）：模拟 1F1B、Interleaved 1F1B 或 ZB1P 等调度策略下的前 / 后向执行时序。
      • 气泡分析（Bubble Analysis）：精确计算因流水线依赖与负载不均衡导致的设备空转时间（Pipeline Bubble）。

    • 3.2.2 依赖与同步（Dependency & Synchronization）

      • 梯度同步机制：模拟反向传播结束后的梯度桶（Bucket）聚合与全局路障（Barrier）。
      • 死锁检测（Deadlock Detection）：在复杂的 P2P 依赖网络与有限的虚拟通道（VC）资源中检测潜在的环路依赖。

    • 3.2.3 训练显存模拟（Training Memory Simulation）

      • 峰值显存追踪：模拟激活值（Activation）、权重梯度与优化器状态的显存占用波峰。
      • 动态优化策略：模拟重计算（Recomputation）带来的额外算力开销，以及 CPU/NVMe 卸载（Offloading）带来的总线带宽压力。

    推理场景仿真：异步事件驱动编排 ¶

    定位：基于 3.1 的映射规则，模拟推理场景下异步、随机性、时延敏感的执行流。

    • 3.3.1 请求级调度（Request-level Scheduling）

      • 到达过程模拟：基于到达过程模型（Arrival Process Models）（如 Poisson Process / MMPP / Self-similar Arrival Process）或真实 Trace 重放，生成随机请求流。
      • 动态批处理（Continuous Batching）：模拟 Orca/vLLM 机制，动态决定何时合并新请求（Iteration-level Scheduling）与抢占逻辑。

    • 3.3.2 阶段性执行流（Phased Execution Flow）

      • Prefill 阶段（Compute-bound）：模拟 Prompt 处理阶段的高算术强度计算，主要受限于 Tensor Core 算力。
      • Decode 阶段（Memory-bound）：模拟逐 Token 生成阶段的访存密集型计算，主要受限于 HBM 带宽。

    • 3.3.3 显存与 KV Cache 管理

      • 分页显存模拟（Paged Memory）：模拟 PagedAttention 机制下的物理块分配、逻辑映射与碎片化效应。
      • 缓存驱逐与交换：模拟显存不足时的 Block Eviction 与 Swap-out/Swap-in 延迟带来的长尾效应。

    资源建模与性能资产库（Resource Modeling & Asset Library）¶

    核心职能：构建高保真的性能资产库，为系统仿真提供原子级性能数据支撑。

    性能资产库构建（Performance Asset Library）¶

    • 资产分级体系：解析模型（Level 1） / 实测画像（Level 2） / 细粒度仿真生成（Level 3）。
    • 资产构建流程：离线微基准测试（Profiling）与数据拟合方法。

    计算资源建模 ¶

    • 通用核心与加速器：对 Tensor Core / Vector Unit 的抽象建模。
    • 执行延迟模型：基于指令流或算子粒度的耗时查询模型。

    存储体系建模 ¶

    • 层次化存储：HBM / SRAM / DDR 的带宽约束与访问延迟模型。
    • 并发访问仲裁：多请求下的总线争抢与排队效应。

    互联网络建模 ¶

    • 拓扑与路由：物理拓扑（Fat-tree/Torus）与逻辑互联的映射。
    • 流级与包级建模：基于流（Flow-level）的带宽分配 vs. 基于包（Packet-level）的拥塞模拟。

    核心应用与挑战 ¶

    典型应用实践 ¶

    • 系统架构设计：下一代 AI 芯片规格定义与集群网络配比优化。
    • 容量规划与采购：基于 SLA 约束的最小硬件规模测算。
    • 自动化策略搜索：复杂并行策略空间中的帕累托最优解（Pareto Optimality）寻优。

    关键挑战与展望 ¶

    • 精度与效率的权衡：多分辨率建模（Multi-resolution Modeling）技术的应用。
    • 仿真可信度构建：物理实测数据与仿真结果的校准（Calibration）闭环。
