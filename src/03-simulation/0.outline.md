## 《AI大模型训练与推理建模仿真白皮书》 大纲

1. 建模仿真概述
   - 背景：大模型时代为何需要负载级建模
   - 仿真方法论：层次化与模块化
   - 负载建模 （第 2 节）
   - 系统建模（第 3 节）
   - 资源建模 （第 4 节）
     - 计算模型 （第 4.1 节）
     - 存储模型 （第 4.2 节）
     - 网络模型 （第 4.3 节）
     - 时钟级 / 功能级 / 事件驱动 / 分析级 
   - 核心应用和关键挑战 （第 5 节）

2. 工作负载建模（Workload Modeling）
   - 仅定义应用需求（L0）
   - 基于DAG的负载描述
   - 不涉及系统与执行

3. 负载驱动的系统级仿真（System-level Simulation）
   - 以负载为输入，将工作负载**映射**到具体资源上
   - 以“训练 / 推理”为主要场景划分

   3.1 训练负载驱动的系统仿真
       - 计算 / 内存 / 通信在训练中的协同
       - CCL、依赖关系、MFU

   3.2 推理负载驱动的系统仿真
       - 请求到达过程
       - TTFT / TPOT / Tail latency

4. 资源级建模方法（Resource-level Modeling Methods）
   - 不再区分训练/推理
   - 作为“工具箱”

   4.1 计算建模
      - 时钟级（GPGPU-SIM）
      - 功能级（Cmodel / SystemC）
      - 分析建模（Roofline）

   4.2 存储建模
      - 层次架构（SRAM / HBM / DDR / NVMe）
      - 仿真器（Ramulator / DRAMSim3 / NVMain）
      - 分析模型（Effective Bandwidth / Latency / Capacity Constraint）

   4.3 网络建模
      - 拓扑（Fat-tree / Dragonfly / Torus）
      - 仿真器（ns-3 / garnet / OMNeT++）
      - 分析模型（bandwidth / delay）

5. 负载级仿真的关键挑战与应用
   - 仿真加速
   - 精度 vs 效率
   - 应用：系统设计 / 容量规划 / 策略评估
