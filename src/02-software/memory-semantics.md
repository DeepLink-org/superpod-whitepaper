# 内存语义的实现与使用

> 产出：硬件系统设计参考与异构分布式共享内存编程规范。

## 互联层的内存语义

- Scale-Up 协议需暴露读写/原子/屏障等事务语义，并明确缓存一致性与完成通知模型。
- 地址全局化：全局物理地址由 Fabric Manager 分配，页表 PTE 携带目标 GPU ID 供交换芯片路由 [@nvidia_fabric_manager]。
- 目标是“通信即访存”：读/写/原子直接在 HBD 内完成，CPU 内核退出数据路径，只做控制平面。

## GPU 的内存一致性

- GPU 内部一致性模型通过 NVLink/Fabric 扩展：UVA + 多级页表使远程显存透明可寻址。
- 原子/订购与屏障需在协议层保证可见性，避免跨设备的乱序与失效。

## Scale-Up 域的统一编址

- UVA/全局物理地址映射在 NVSwitch 域扩展为全局 PA，硬件完成地址翻译与路由，不再依赖 CPU 作为数据路径。
- IOMMU/权限：稀疏地址空间、分级访问控制与隔离域是共享内存安全性的前提。
- 访问链路的故事版：驱动为远程 GPU 分配对等 ID → 页表把 VA 指向全局 PA → MMU 翻译得到目标 GPU → NVSwitch 路由到对端显存，完成如同本地的读写。

## 现代 GPU 访存体系（节点内）

- GPU 通过 UVA 将显存/主存映射到统一虚拟地址，页表支持多级映射与 4K–128K 页。  
- 节点内 P2P：驱动分配 Peer ID，页表标记 Peer，MMU 翻译后直接经 NVLink/PCIe 访问对端显存。  
- 内核只在资源管理时介入，数据路径由硬件直接完成。

## 超节点访存体系（跨节点）

- 传统 PCIe P2P 受限于单一 Root Complex，无法跨物理节点统一访存。  
- 超节点引入 **全局物理地址空间**：Fabric Manager 划分 47-bit 全局 PA 段，PTE 标记目标 GPU。  
- GPU 访问 VA → MMU 翻译出全局 PA + 目标 GPU → NVSwitch/Fabric 按地址路由到对端显存，CPU 不再参与数据面。  
- 价值：跨节点统一寻址 + 低延迟读写，为单边通信与共享内存编程奠定硬件基础。
