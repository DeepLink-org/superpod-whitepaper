# 训练与推理系统设计与优化

## 训练案例

- Cloud Matrix 384 训练优化案例（占位）：目标 300 PFLOPS FP16，需验证并行策略、通信重叠与 Checkpoint 容错。
- 并行组合：数据并行 + 张量并行 + 管道并行的平衡，结合拓扑感知的作业编排。
- 训练效率：覆盖率/利用率指标、Straggler 缓解、带宽/算力配比优化。
- 故事线：HBD 内近线速通信让梯度同步成本下降，但跨 HBD 仍需收敛比与分层调度；Checkpoint 时要兼顾 NVLink/HBM 与 PCIe/网络 IO 的平衡。
- 背景引用：NVLink 4.0 带宽 900 GB/s vs PCIe 5.0 x16 128 GB/s，跨域差距 7×，训练作业需布局“域内近线速 + 域间收敛比”策略。

## 推理案例

- Cloud Matrix 384 推理优化案例与异构 PD 推理（占位）：关注路由/批合并、KV/权重缓存、调度策略。
- 延迟与成本：TP99 目标、混合精度（FP8/FP6/FP4）与稀疏路由带来的成本/延迟平衡。
- 叙事化：长序列/多模态推理中，不同请求时长差异大，批合并与路由策略决定尾延迟；KV/权重缓存放置（HBD 内/跨 HBD）决定成本。
