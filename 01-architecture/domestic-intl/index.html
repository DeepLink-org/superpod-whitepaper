    国内外先进算力架构分析¶

    现状与趋势¶

    • 超节点热度高企：NVIDIA 宣称 NVL72 提供 1,440 PFLOPS，华为 CloudMatrix 384 标称 300 PFLOPS FP16（对标 GB200 NVL72 的 ~180 PFLOPS），各云厂与国产 GPU 厂在协议上同时开辟新品牌（UALink、ALS、Eth-X、OSIA、EthLink 等），一场“互联军备竞赛”全面开启。
    • NVIDIA 以 NVLink/NVSwitch 主导的 Scale-Up 域持续放大：NVL32 → NVL72，再到 Rubin 世代规划的 144 超级芯片互联，单套算力从 32 PFLOPS（NVL32）迈向 1.4 EFLOPS 级别 [@nvidia_gb200; @nvidia_blackwell_ultra]。
    • 竞争焦点从“堆 GPU”转向“互联 × 封装 × 内存 × 编程模型”的系统协同；Scale-Up 域成为各家抢占的主战场，缺乏系统性视角容易在利用率和扩展效率上遭遇“崖降”。

    生态差异要点¶

    • 封装与 D2D：NVIDIA NV-HBI（双 Die，10 TB/s 级）把 D2D 作为算力放大的核心；国内方案在 CoWoS/EMIB 类 2.5D/3D 形态快速追赶。
    • 互联协议：NVIDIA 坚持 NVLink 专用协议，国内/云厂商尝试“以太化”或 CXL 兼容的 Scale-Up 协议以降低生态摩擦。
    • 软件栈：CUDA/NCCL 在集合通信与编程体验上仍优势明显；国内需在 PGAS/单边通信与工具链上补齐。

    叙事视角：系统化而非“堆 GPU”¶

    • 从 Volta/HGX 到 Blackwell/GB200，算力增长源自低精度格式（~16×）、Tensor Core（~12×）、工艺（~2.5×）与互联/封装协同的复合放大 [@huangs_law]。NVIDIA 的战略是“系统即产品”，每两年提供 ≈6× 系统算力、十年追求 7000× 乃至万倍。
    • 超节点的核心价值在于把“互联-内存-封装-编程模型”联动优化，形成机柜级 HBD，避免 Scale-Out 单纯靠以太/IB 导致的通信/一致性瓶颈。国内外竞争者若缺失系统性策略，易在实际大模型训练/推理中遭遇效率折损。

    什么是超节点（重述）¶

    超节点通过高速无损互联突破传统“CPU+PCIe”节点边界，在机柜级构建高带宽域（HBD），让多 GPU/NPU 像一张“超级 GPU”一样工作。硬件层面用 NVLink/CXL/专有交换实现低延迟直连；软件层面绕过内核协议栈，主要依赖用户态通信库（NCCL/HCCL 等）调度，显著降低开销。

    • 硬件互联：NVLink/NVSwitch 或同类高速互联把数十/上百 GPU 织成低延迟域，远高于 PCIe/以太。
    • 统一寻址：UVA + GPU MMU + 全局物理地址，让远程显存像本地一样可寻址，Fabric 负责路由。
    • 软件范式转变：从 CPU 调度的双边通信，转向 GPU 主导的单边通信与共享内存语义，CPU 更多做控制平面。

    系统分层蓝图（文字版）¶

    1. Die 内：PE 通过 NoC 互联。
    2. Chip 内：多 Die 通过 D2D（如 NV-HBI/UCIe）互联成一颗 GPU/NPU。
    3. Node 内：多 Chip 通过 C2C（NVLink+NVSwitch/PCIe/CXL）互联。
    4. SuperPod/HBD：节点间以 NVSwitch Fabric/自研协议组成机柜级 Scale-Up 域。
    5. Cluster：SuperPod 之间通过 RDMA/以太/IB 进行 Scale-Out。

    真正的竞争点在于第 3-4 层：如何在机柜级保持低直径、高二分带宽与可重构弹性，同时让编程模型保持简单可用。
